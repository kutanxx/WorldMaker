# 성 밖 교외(Faubourg) Phase 1 — 설계

작성일: 2026-07-01
상태: 설계 확정 (구현 계획 작성 전)
대상: 도시 드릴다운(유기적 형태 + 바이오미, 이미 merged) 확장. 성벽 밖 교외를 추가.

## 1. 목표

사용자 지적: "건물이 꼭 성 안에만 있는 건 아니다." 지금 도시는 모든 것을 boundary(=성벽 선) 안으로 잘라내 성 밖이 텅 비어 있다. 중세 도시는 성벽 밖에 **교외(faubourg)** 가 있었다.

**자료 조사 결과(근거):** 교외는 성문에서 뻗은 도로를 따라 형성됐고 그 도로 이름을 따랐을 만큼 "진입로 리본"이 정체성. 성문 근처(시장 옆)가 밀집, 밖으로 갈수록 성김. 성벽 없는 비보호 지대(반농반도). 혐오·화재 위험 업종(무두질·도살·유리)은 성 밖으로. 강가 물레방아·언덕 풍차 등도 성 밖.

**Phase 1 범위(사용자 선택):** ① 교외 리본 + ② 소수 구조물(물레방아/풍차). 밭·과수원·특수시설(묘지·교수대)·바이오미 특화 교외는 다음.

## 2. 핵심 설계 결정 (자가 피드백에서 도출)

- **viewBox `0 0 300 300` 유지**, 도시 크기·바다 로직 불변. (viewBox 확장은 가장자리 바다 띠 렌더가 깨져 폐기.)
- 교외는 **기존 boundary→캔버스 가장자리 여백 안에만** 기회주의적으로 배치. 여백 계산: boundary 반경 ≈ `(58+size*12)*(0.8~1.14)`, 중심 (150,150), 캔버스 반경 150 → size 4 여백 ~29~65px(리본 2~3채), 큰·선형 도시는 여백이 좁거나 없음.
- **한계(정직):** 큰·선형 도시는 교외가 성기거나 거의 없음. 히스토리(큰 도시=큰 교외)와 반대지만, "프레임 꽉 채운 대도시 + 소도시엔 또렷한 교외"로 시각적으론 자연스러움. 도시 축소로 여백 확보는 별도 후속.
- 교외는 **boundary clip 밖 별도 그룹**으로 렌더(성 밖에 보이게). viewBox가 캔버스 밖을 자동으로 잘라줌.
- 결정성: 교외 rng는 **성내 생성이 모두 끝난 뒤 고정 지점**에서 소비 → 성내 레이아웃 불변.

## 3. 데이터 모델 — `CityLayout` 추가

```ts
suburbRoads: Polyline[];   // 성문 밖으로 연장한 진입로
suburbs: Polygon[];        // 교외 건물(리본) 풋프린트
outworks: { type: "watermill" | "windmill"; at: Point; angle: number }[];
```

`Point`/`Polygon`/`Polyline`은 `engine/geometry.ts`.

## 4. 생성 (city.ts)

`generateCityLayout`에서 성벽·성문(`wall.gates`)·성내 건물이 모두 계산된 뒤, 고정 rng 지점에서:

**진입로 + 리본(성문별 기회주의):**
- 대상 성문: `wall.gates` 중 최대 `1 + floor(size/2)`개(캔버스 여백 있는 것 우선).
- 각 성문 g에서 바깥 방향 `dir = normalize(g - center)`. 진입로: g에서 `dir`로 시작점을 `+8`(성벽·해자 밖) 떨어뜨려, 길이 `L`(≈ 여백에 맞춰 `min(38, 캔버스까지 거리 - 6)`)만큼 연장. 약간의 측방 굴곡(노이즈 소량). 진입로가 물에 들어가거나 캔버스 밖이면 그 성문은 스킵.
- 리본 건물: 진입로를 따라 등간격(≈9px)으로 좌우 수직 오프셋 `±(4~8)`에 작은 풋프린트(≈5×4). 문에서 멀수록 배치 확률 감소(예: `p = 0.9 - t*0.5`). 조건: **`!pointInPolygon(boundary)` && `!inWater(water)` && 캔버스 안(여유 3px) && 다른 교외 건물과 6px 이상**.

**구조물(1~2개):**
- boundary 밖 + 캔버스 안에서 물에 인접한 지점이 있으면 `watermill`(그 지점), 없으면 외곽 빈 땅에 `windmill`. `angle`은 렌더용(풍차 날개/물레 방향), rng로 결정적.

성내 도시가 작거나(여백 없음) 성문이 없으면 `suburbs`/`suburbRoads`/`outworks`는 빈 배열.

## 5. 렌더 (svgCityRenderer)

성내 `clipped` 그룹(및 그 clipPath) 밖에, **비클립 교외 그룹**을 물 다음·성내 그룹 앞에 추가:
- `.suburb-road`: 가늘고 흐린 흙길(메인 도로보다 얇게, 예 `#c9bb96` w1.6).
- `.suburb`: 작은 집(성내보다 작고 성기게, 예 `#e0d6c0` stroke `#9a8a70`).
- `.outwork`: `watermill`=작은 집+물레바퀴 원, `windmill`=기둥+날개 십자 글리프(`at`,`angle` 사용).
viewBox가 캔버스 밖을 자르므로 별도 clip 불필요. 범례·제목은 최상위(마지막)라 겹치면 덮음.

## 6. 아키텍처

| 모듈 | 작업 |
|---|---|
| `city.ts` | 교외 생성(진입로·리본·구조물) + `CityLayout` 3필드. 재사용: `bbox`·`pointInPolygon`(geometry), `inWater`(water), `wall.gates`, `centroid`. |
| `svgCityRenderer.ts` | 비클립 교외 그룹(진입로·집·구조물) 렌더 분기. |

새 모듈 없이 city.ts에 응집(작음). 공개 시그니처(`cityContext`/`generateCityLayout`/`renderCity`) 유지 → `app.ts` 무변경.

## 7. 테스트 (스크린샷 막힘 → DOM 지표)

- 여백 있는 중형 내륙 도시(예 size 4, 초원, 적절 시드): `suburbs.length > 0`; 모든 교외 건물이 **boundary 밖 + 물 밖 + 캔버스 안**; 진입로가 성문 근처에서 시작.
- 아주 작은 도시 또는 여백 없는 경우: 교외 거의/전혀 없음(크래시 없이 빈 배열).
- `outworks`: 존재 시 boundary 밖 + 캔버스 안; watermill이면 물 인접.
- 결정성: 같은 (ctx,seed) 동일; 성내 레이아웃이 교외 추가로 안 바뀜(교외 rng가 끝에서 소비됨을 간접 확인).
- 렌더 jsdom 스모크: `.suburb`/`.suburb-road`/`.outwork`가 **비클립 그룹**에 존재; 개수 일치.

## 8. 위험 / 범위 밖

- 여백이 좁아 큰·선형 도시 교외가 소박(고유 트레이드오프, §2). 배치 실패 시 빈 배열(견고).
- 시각 튜닝(리본 밀도·집 크기·구조물 글리프)은 스크린샷 막힘 → DOM + localhost 육안.
- **범위 밖(다음):** 밭·과수원, 특수시설(묘지·교수대), 바이오미 특화 교외(사막 카라반세라이/늪 둑길 교외), 교외 2차 성벽, 도시 축소로 교외 공간 확보.
